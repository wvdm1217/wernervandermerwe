Due to my growing need for compute and to over engineer I decided to start running my own [Kubernetes](https://kubernetes.io/) cluster. 
There is a growing push towards being able to self host the software you depend on. 
Kubernetes has been at the centre of this movement for many. 

My hope is that there is something in what I did that inspires you to self host your own apps. 
Or at the very least you tell me what I did wrong and I can improve things even more.
There is a lot of yaml and opinions included.

*This post will probably change over the years as I add or amend my setup. The bottom of the post includes a changelog and future updates.*

## Overview 

The core of the stack includes,
- An Ubuntu server running K3s and setup using Ansible.
- ZeroTier for remote SSH access to the server.
- ArgoCD for deploying Helm charts using GitOps.
- Cloudflare Tunnels and Traefik for setting up routes to apps in the cluster.
- External Secret Manager to store secrets in Google Cloud Secret Manager.
- Grafana + Prometheus for Observability.

## Server Setup

As a server OS I flashed a headless [Ubuntu Server](https://ubuntu.com/tutorials/install-ubuntu-server) onto a 16GB machine.

This server is currently the only node and a good starting point for a simple Kubernetes cluster.
People often virtualise compute using tools like [Proxmox](https://www.proxmox.com/).
For simplicity I am running the server bare metal.

The following is going to be installed on the server,
- k3s
- Helm
- ArgoCD
- ZeroTier

### K3S

Kubernetes has different distributions depending on your needs. 
I selected [K3s](https://k3s.io/) due to being lightweight and not needing the complexity that comes from a more complete distribution.
This seemed to keep the memory consumption, my main bottleneck to running more useless apps, low.

### Ansible 

All updates to the server is done using [Ansible](https://docs.ansible.com/projects/ansible/latest/index.html) to get reproducible setups.
This allows me to wipe or replace servers and not have to start from scratch.
There is also a very nice collection of Ansible playbooks for k3s, https://github.com/k3s-io/k3s-ansible.

#### Inventory

The single node server needs to be added to the inventory as follows,

```ini
# inventory.ini
[homelab_servers]
homelab ansible_host=YOUR_HOST_IP ansible_user=YOUR_USERNAME
```

Next the config file needs setup to install k3s.


```yaml
# inventory.yaml
# The original sample file, https://github.com/k3s-io/k3s-ansible/blob/main/inventory-sample.yml.

k3s_cluster:
  children:
    server:
      hosts:
        192.168.1.1:

  vars:
    ansible_port: 22
    ansible_user: user
    k3s_version: v1.33.1+k3s1
    token: "changeme!" # Keep this safe.
    api_endpoint: "{{ hostvars[groups['server'][0]]['ansible_host'] | default(groups['server'][0]) }}"
    extra_server_args: "--disable traefik"
    user_kubectl: true
```

The config specifies the version of k3s to install and also to disable the Traefik plug in.
We instead deploy Traefik later via GitOps.

1. **Test connectivity:**
   ```bash
   ansible all -i inventory.ini -m ping
   ```

2. **Install K3s:**
   ```bash
   ansible-galaxy collection install git+https://github.com/k3s-io/k3s-ansible.git
   ansible-playbook k3s.orchestration.site -i inventory.yml --ask-become-pass
   ```

If all went well you now have a kubernetes cluster running!
This is the moment that all nerds dream of. 

#### Helm

[Helm](https://helm.sh/) is used to manage kubernetes applications.
We need helm in order to install ArgoCD onto our cluster and any other manual installations you would want before making use of GitOps.

**Install Helm Playbook:**

```yaml
# playbooks/install-helm.yaml
- name: Install Helm on homelab servers
  hosts: homelab
  become: yes
  tasks:
    - name: Download Helm installation script
      get_url:
        url: https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3
        dest: /tmp/get_helm.sh
        mode: '0700'

    - name: Install Helm using official script
      shell: /tmp/get_helm.sh
      args:
        creates: /usr/local/bin/helm

    - name: Verify Helm installation
      command: helm version --short
      register: helm_version
      changed_when: false

    - name: Display Helm version
      debug:
        msg: "Helm installed: {{ helm_version.stdout }}"

    - name: Clean up installation script
      file:
        path: /tmp/get_helm.sh
        state: absent
```

Install the playbook as follows,

```sh
ansible-playbook -i inventory.ini playbook/install-helm.yml --ask-become-pass
```

You can verify that helm exists on the server by running,

```sh
helm version
```

#### ArgoCD

We also install ArgoCD with an Ansible playbook. 
Thereafter all our applications are managed by ArgoCD.
We will configure ArgoCD in a subsequent step. 

**Install ArgoCD:**

```yaml
---
- name: Install ArgoCD on K3s cluster
  hosts: homelab
  become: yes
  tasks:
    - name: Install required Python packages
      apt:
        name:
          - python3-pip
          - python3-kubernetes
        state: present
        update_cache: yes

    - name: Create argocd namespace
      kubernetes.core.k8s:
        name: argocd
        api_version: v1
        kind: Namespace
        state: present
        kubeconfig: /etc/rancher/k3s/k3s.yaml

    - name: Add ArgoCD Helm repository
      kubernetes.core.helm_repository:
        name: argo
        repo_url: https://argoproj.github.io/argo-helm
        kubeconfig: /etc/rancher/k3s/k3s.yaml

    - name: Install ArgoCD using Helm
      kubernetes.core.helm:
        name: argocd
        chart_ref: argo/argo-cd
        release_namespace: argocd
        create_namespace: true
        kubeconfig: /etc/rancher/k3s/k3s.yaml
        values:
          server:
            service:
              type: NodePort
              nodePortHttp: 30080
              nodePortHttps: 30443
          configs:
            params:
              server.insecure: true
          metrics:
            enabled: true

    - name: Wait for ArgoCD server to be ready
      kubernetes.core.k8s_info:
        api_version: apps/v1
        kind: Deployment
        name: argocd-server
        namespace: argocd
        kubeconfig: /etc/rancher/k3s/k3s.yaml
        wait: true
        wait_condition:
          type: Available
          status: "True"
        wait_timeout: 300

    - name: Get ArgoCD admin password
      kubernetes.core.k8s_info:
        api_version: v1
        kind: Secret
        name: argocd-initial-admin-secret
        namespace: argocd
        kubeconfig: /etc/rancher/k3s/k3s.yaml
      register: argocd_secret

    - name: Display ArgoCD access information
      debug:
        msg: |
          ArgoCD is now installed!
          Access URL: http://{{ ansible_host }}:30080
          Username: admin
          Password: {{ argocd_secret.resources[0].data.password | b64decode }}
      when: argocd_secret.resources | length > 0
```

We expose ArgoCD using a NodePort at port `30080`. 

```sh
ansible-galaxy collection install kubernetes.core
ansible-playbook -i inventory.ini playbooks/install-argocd.yml --ask-become-pass
```

The password that is fetched can be stored and used as admin credentials to the ArgoCD UI.
